Product Sentiment Analysis Engine

Multi-source review scraping → custom NLP pipeline → attribute-level sentiment classification → actionable quality insights for PU & MM teams.

Customer voice is scattered across e-commerce platforms, brand websites and internal CRM logs. This project consolidates all review channels, extracts structured opinion signals and routes quality feedback directly to production and material management teams — closing the loop between end consumer and factory floor.

Problem Statement

Returns and quality complaints about footwear products were escalating without a structured feedback mechanism from the market. Attribute-level issues (material, MRP perception, comfort, elasticity) were buried inside unstructured review text and could not be systematically quantified or prioritised.

Data Sources & Scraping Stack

Four distinct platforms were scraped using a combination of Selenium WebDriver (for JavaScript-rendered pages) and BeautifulSoup4 (for static HTML parsing).

• Flipkart — product pages, star ratings, verified buyer reviews, images

• Amazon India — reviews, Q&A sections, review helpfulness scores

• Company website — official product pages and review widgets

• Internal cloud CRM portal — dealer complaints and end-customer tickets

Sentiment Classification Approach

A two-stage pipeline was employed: a coarse polarity classifier followed by an aspect-based sentiment analysis (ABSA) layer that assigns sentiment to five product attributes.

Stage 1 — Polarity Classifier (TF-IDF + Logistic Regression)

Reviews are first classified as Positive / Neutral / Negative using TF-IDF vectorisation combined with a regularised Logistic Regression.

TF-IDF(t, d) = TF(t, d) × log( N / df(t) ) where: TF(t,d) = frequency of term t in document d N = total number of documents df(t) = number of documents containing term t
Term Frequency–Inverse Document Frequency weighting
P(y = k | x) = exp(β_k · x) / Σ_j exp(β_j · x) Loss: L(β) = -Σ log P(y_i | x_i) + λ||β||²
Multinomial logistic regression with L2 regularisation

Stage 2 — Aspect-Based Sentiment Analysis

Five product attributes were defined as extraction targets. Aspect terms were matched using a domain lexicon and dependency parse trees (spaCy), then a fine-tuned BERT-based classifier (bert-base-multilingual-uncased) was used for aspect-sentiment assignment.

• Material — sole quality, stitching, fabric durability

• MRP / Value — price perception, value for money

• Comfort — cushioning, arch support, fit

• Elasticity — stretch, flexibility, resilience

• Aesthetics — design, colour, finish

BERT Input: [CLS] review_text [SEP] aspect_term [SEP] Output: sentiment ∈ {Positive, Neutral, Negative} Fine-tune objective: L = CrossEntropy( softmax(W · h_[CLS]), y_true )
BERT sentence-pair classification for ABSA

Star-Based Category Segmentation

• 4–5 stars → Promoter segment (positive reinforcement signals)

• 3 stars → Neutral segment (latent issues, requires NLP mining)

• 1–2 stars → Detractor segment (priority quality flag)

Sales Demand Forecasting System

SKU-level time-series forecasting using XGBoost + SARIMA ensemble to optimise production scheduling and inventory positioning.

Overview
Feature Engineering
Model Design
Results
Business Impact

Footwear demand is highly seasonal, geographically heterogeneous and influenced by festival calendars, monsoon cycles and regional trade patterns. This model forecasts SKU-level weekly demand 4–8 weeks ahead across 18 regional zones, enabling proactive production planning and safety stock computation.

Business Problem

Production batches were planned on static historical averages, leading to overstock in slow-moving SKUs and stockouts in fast-moving categories during festival and back-to-school peaks. The objective was to reduce both overstock waste and lost sales simultaneously.

18 Regional Zones
4–8 wk Forecast Horizon
8.3% MAPE (best SKU tier)
~22% Overstock Reduction

Data Sources

• SAP HANA — 36 months of weekly SKU-level sales by zone and dealer

• Production logs — batch sizes, cycle times, mould availability

• Festival & holiday calendar — regional event flags

• Weather API — monsoon onset dates by region (footwear is weather-sensitive)

• Pricing table — MRP changes and promotional discounts history

Feature Engineering

• Lag features: sales at t-1, t-2, t-4, t-8 (weekly lags)

• Rolling statistics: 4-week rolling mean, std, min, max

• Calendar encoding: week-of-year, month, quarter, is_festival_week

• Trend decomposition: STL decomposition → trend, seasonal, residual components as features

• Zone-SKU interaction: historical zone preference index per SKU category

• Price elasticity proxy: (MRP_current - MRP_baseline) / MRP_baseline

Model Architecture — Ensemble Approach

Two complementary models are trained per SKU-zone combination and blended via weighted average, where weights are tuned on a held-out validation window.

Component 1 — SARIMA (Statistical Baseline)

SARIMA(p, d, q)(P, D, Q)[S] Non-seasonal: AR(p) × I(d) × MA(q) Seasonal: AR(P) × I(D) × MA(Q) at period S = 52 (weekly) ARIMA equation: Φ_P(B^S) φ_p(B) ∇^d_S ∇^d y_t = Θ_Q(B^S) θ_q(B) ε_t Parameters selected by minimising AIC: AIC = 2k − 2 ln(L̂)
Seasonal ARIMA — captures trend and seasonal structure

Component 2 — XGBoost Regressor (Gradient Boosting)

Objective: L(θ) = Σ_i l(y_i, ŷ_i) + Σ_k Ω(f_k) where Ω(f) = γT + ½λ||w||² T = number of leaves w = leaf weights γ, λ = regularisation hyperparameters Gradient boosting update: ŷ_i^(t) = ŷ_i^(t-1) + η · f_t(x_i) η = learning rate (tuned: 0.05) max_depth = 6, n_estimators = 500, subsample = 0.8
XGBoost with L1/L2 regularisation — captures non-linear feature interactions

Ensemble Blend

ŷ_final = α · ŷ_SARIMA + (1 − α) · ŷ_XGBoost α optimised on validation set using grid search: α* = argmin_α RMSE(y_val, α·ŷ_SARIMA + (1-α)·ŷ_XGB) Typical optimal range: α ∈ [0.25, 0.45]

Customer Churn Prediction Model

Behavioural classification model to identify high-risk customers before they disengage — enabling targeted retention campaigns.

Overview
Feature Design
Model
Results
Impact

End-consumers who purchase through company-owned digital channels (website, cloud portal) exhibit measurable behavioural signals before they stop re-purchasing. This model scores each active customer on a monthly churn probability, enabling the marketing team to prioritise retention spend.

Churn Definition

A customer is labelled churned if they have made no purchase within the last 90 days AND had previously purchased at least twice within the preceding 180-day window. This definition avoids penalising one-time buyers.

~41K Customer Records
17.3% Churn Rate (baseline)
0.88 AUC-ROC
0.82 Precision@Top 20%

RFM + Behavioural Feature Matrix

Features are constructed around the classical RFM (Recency, Frequency, Monetary) framework, extended with engagement and product affinity signals.

RFM Features

• Recency (R): days since last purchase

• Frequency (F): number of orders in 6 months

• Monetary (M): total spend (INR) in 6 months

Extended Signals

• Category breadth (# distinct categories bought)

• Average inter-purchase interval (days)

• Return / refund count ratio

• Promotion response flag

Class Imbalance Handling

The dataset is imbalanced (~17% churn). SMOTE (Synthetic Minority Over-sampling Technique) was applied on the training fold only to avoid data leakage.

SMOTE interpolation: x_new = x_i + λ · (x̃_i − x_i), λ ~ Uniform(0, 1) where x̃_i is a randomly selected k-nearest neighbour of x_i in the minority class feature space.

Model — Gradient Boosted Trees (XGBoost)

XGBoost was selected after benchmarking against Logistic Regression, Random Forest and LightGBM. It produced the best precision-recall trade-off on the validation set.

Binary classification objective (log-loss): L = -[y log(p) + (1−y) log(1−p)] where p = σ(Σ_k f_k(x)) and σ(z) = 1 / (1 + e^{-z}) Regularised objective: L(θ) = Σ_i l(y_i, ŷ_i) + Σ_k [γT_k + ½λ||w_k||²] Hyperparameters (tuned via Optuna): learning_rate = 0.04 max_depth = 5 subsample = 0.75 colsample_bytree= 0.80 scale_pos_weight= 4.8 (handles class imbalance)

SHAP Explainability

SHAP (SHapley Additive exPlanations) values were computed to provide feature attribution per customer, making the model auditable by the marketing team.

SHAP value for feature j: φ_j = Σ_{S ⊆ F\{j}} [ |S|!(|F|−|S|−1)! / |F|! ] · [f(S∪{j}) − f(S)] Key interpretation: φ_j > 0 → feature j pushes prediction toward churn φ_j < 0 → feature j pushes prediction toward retention

Model Evaluation

0.88AUC-ROC
0.84F1 (churned class)
0.82Precision

Precision = TP / (TP + FP) Recall = TP / (TP + FN) F1-Score = 2 · (Precision · Recall) / (Precision + Recall) AUC-ROC = P(score(churn) > score(non-churn))

Top Churn Drivers (SHAP importance)

1. Recency — days since last purchase (highest impact)

2. Inter-purchase interval trend (increasing gap = high risk)

3. Return/refund ratio (high returns precede churn)

4. Frequency decline (3-month vs 6-month frequency ratio)

5. Category breadth (single-category buyers churn faster)

Marketing Activation

• Monthly scored list delivered to CRM for targeted re-engagement

• Top 20% high-risk segment enrolled in personalised coupon campaign

• Model explanation (SHAP summary) used in campaign messaging strategy

• Pilot re-engagement campaign achieved 23% recovery rate on high-risk cohort

Dealer Churn & Disengagement Prediction

Binary classification model to identify at-risk dealers before order volume drops — protecting the B2B distribution network.

Overview
Features
Model
Results
Impact

Dealer relationships are the backbone of Walkaroo's distribution network. A dealer who progressively reduces order frequency or stops reordering represents a significant revenue risk and potential market share loss in their territory. This model identifies early disengagement signals at the dealer level, enabling the sales team to intervene before attrition occurs.

Dealer Churn Definition

A dealer is flagged as at-risk if their 3-month order volume has declined by more than 30% relative to their 12-month baseline, OR if they have had zero orders in the past 45 days despite being active (≥ 4 orders) in the prior 6 months.

~3,200Active Dealers
11.8%Churn Rate (annual)
0.91AUC-ROC
6 wksAvg Early Warning Lead

Dealer Behaviour Features

• Order cadence: avg days between consecutive orders (rolling 90-day)

• Volume trend: linear slope of monthly order quantity over 6 months

• SKU diversity: number of distinct product lines ordered

• Payment behaviour: average days-past-due (DPD) on invoices

• Return rate: returned units / total ordered units

• Complaint frequency: support tickets per 100 units

• Competitor activity proxy: territory-level sales drop vs. region average

• Sales rep visit frequency: logged field visits per quarter

Volume Trend Feature (Linear Regression Slope)

For dealer d, monthly order volume {v_1, v_2, ..., v_6}: Fit OLS: v_t = β_0 + β_1·t + ε_t Trend feature = β̂_1 (slope coefficient) β̂_1 = [Σ(t - t̄)(v_t - v̄)] / [Σ(t - t̄)²] Interpretation: β̂_1 < -threshold → declining engagement signal β̂_1 > 0 → growing dealer
OLS trend slope as a structured churn signal

Model — Random Forest + Logistic Regression Stacking

A stacked ensemble combines a Random Forest (base model) with a Logistic Regression meta-learner. The meta-learner is trained on out-of-fold predictions from the base model.

Level-0 (Base Model — Random Forest): ŷ_RF = (1/T) Σ_{t=1}^{T} f_t(x) Each tree f_t trained on bootstrap sample. Gini impurity split criterion: G(t) = 1 − Σ_k p_k² Level-1 (Meta-learner — Logistic Regression): ŷ_final = σ(w · ŷ_RF + b) Trained on out-of-fold (OOF) predictions to prevent overfitting to training distribution.
Cross-validated stacking (5-fold): For each fold k: Train RF on folds ≠ k Predict on fold k → OOF_k Meta-features = [OOF_1, OOF_2, ..., OOF_5] Train LR on full meta-feature matrix
